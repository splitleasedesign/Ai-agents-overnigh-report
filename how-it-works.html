How the AI Design Team Works

A system of 10 specialized AI agents that analyze, audit, and improve the Split Lease platform. Each agent has a unique perspective inspired by legendary designers.

Overview

What is this?

The AI Design Team is a collection of 10 AI agents, each specialized in a different aspect of design and UX. They're powered by Claude and can be triggered to analyze websites, create mockups, write reports, and suggest improvements.

Prompt
→
AI Agent
→
Report / Mockup

Each agent has persistent memory, knows the Split Lease codebase, and builds on previous work. They can work independently or collaborate on complex projects.

Setup

Folder Structure

Each agent has its own folder with three key files. Runs (overnight batch jobs) are organized by date.

AI Team/ ├── agents/ │ └── [agent-name]/ │ ├── agent.md ← Personality, expertise, constraints │ ├── memory.md ← Session logs, lessons learned │ └── prompt.md ← Current task assignment │ ├── runs/ │ ├── run-001--2026-01-29/ │ │ ├── _manifest.md ← What each agent was assigned │ │ └── [agent-name]/ │ │ ├── html/ ← Mockups produced │ │ └── reports/ ← Analysis reports │ │ │ └── run-002--2026-01-30/ │ └── ... │ └── orchestrator/ └── agent.md ← Coordinates all agents

Process

How to Run an Agent

1

Write the prompt

Create or update the agent's prompt.md file with the task. Be specific about what you want analyzed and where to save results.

2

Trigger the agent

In Claude Code, navigate to the agent folder and say: "read your prompt.md". The agent will read its personality (agent.md), check its memory, and execute the task.

3

Review deliverables

The agent saves reports and mockups to the specified run folder. It also updates its memory.md with a session log.

Example Trigger Command

Open Claude Code, then type:

cd agents/don-norman
read your prompt.md

Templates

Prompt Structure

Each prompt.md follows this format:

# Task: [Task Name]
Date: 2026-01-31
Priority: medium
Run: run-003--2026-01-31

## Context
[Background information, what to read first]

## Objective
[Clear description of what to produce]

## Deliverables
Save to: `runs/run-003--2026-01-31/[agent]/reports/[filename].md`

## When Done
1. Save report to specified location
2. Update memory.md with session log:
   `[date] task-name | deliverable | ~tokens | path | feedback | lesson`

Important Distinction

Memory vs Reports

memory.md reports/
Session logs (one line per run) Full deliverables and analysis
Lessons learned Detailed findings
User feedback received Mockups and code
Tips for future sessions Recommendations

Memory Log Format

[2026-01-31] comet-prompts | 6 prompts | ~2k tokens | runs/run-003.../report.md | positive | learned X

Batch Processing

Overnight Runs

For large projects, we trigger multiple agents in sequence. Each run gets a unique ID and date:

Run Date Focus Output
run-001 Jan 29, 2026 Initial platform audit Reports + mockups
run-002 Jan 30, 2026 Deep-dive analysis 26 HTML mockups
run-003 Jan 31, 2026 Comet integration 54 testing prompts

Advanced

Cross-Agent Collaboration

Agents can build on each other's work. Example from Run 003:

Massimo Vignelli
Creates 34 graphic specs
→
Charles Eames
Produces all 34 graphics

Massimo's graphics-strategy.md defines what graphics are needed. Charles reads this and produces each one using his Empty State component approach.

Automation

Comet Testing Prompts

Agents can create prompts for Perplexity Comet — an AI browser that tests websites automatically.

Example Comet Prompt (Don Norman)

TASK: Complete a proposal for non-contiguous days
URL: https://www.splitlease.com/listing/test-123
STEPS:
1. Click "Request to Book"
2. Select Monday and Thursday (non-adjacent)
3. Fill required fields with test data
4. Submit proposal
VERIFY:
- Confirmation modal appears
- Selected days shown correctly
- No console errors
OUTPUT: Pass/Fail with screenshot

Each agent creates prompts for their domain — Don Norman tests usability flows, Jony Ive tests mobile viewports, Dieter Rams verifies trust claims are accurate.
